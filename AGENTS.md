# Workout Tracker — AGENTS.md

## Stack
- **Frontend**: Svelte 5 (runes), Vite 8 (not SvelteKit), Tailwind CSS v4, FlyonUI, Iconify (boxicons)
- **Backend**: Cloudflare Workers (ES module), D1 (SQL), KV (sessions), R2 (images)
- **Auth**: Google OAuth 2.0
- **Language**: TypeScript (strict), Biome (lint/format, no ESLint/Prettier)
- **Package manager**: bun

## Commands
| Command | Action |
|---|---|
| `bun run dev` | Dev server with HTTPS (localhost:5173) |
| `bun run build` | Production build to `dist/` |
| `bun run preview` | Build + `wrangler dev --local-protocol=https` |
| `bun run deploy` | Build + `wrangler deploy` |
| `bun run check` | `svelte-check --tsconfig ./tsconfig.app.json && tsc -p tsconfig.node.json` |
| `bun run biome:check` | Lint + format all files |
| `bun run biome:format` | Format only |
| `bun run typegen` | Regenerate `worker-configuration.d.ts` |
| `bun run deploy:secrets` | Bulk upload secrets from `.env` |

## Architecture
- **Entry (client)**: `index.html` → `src/main.ts` → `src/App.svelte`
- **Entry (worker)**: `worker/index.ts` — single exported `ExportedHandler<Env>` with fetch handler
- **Routing**: Client-side only — `store.currentPage` (`$state`) toggles between `home`/`workout`/`training` views in `App.svelte`
- **Store**: Global reactive store (`src/store.svelte.ts`) using `$state`, persisted to localStorage for user
- **Feature layout**: `src/{home,workout,training}/` — each feature owns its pages + subcomponents

## TypeScript
- Three tsconfigs: `tsconfig.app.json` (src, typechecked by svelte-check), `tsconfig.node.json` (vite.config.ts), `tsconfig.worker.json` (worker/)
- Worker types generated by `wrangler types` into `worker-configuration.d.ts`

## Conventions
- **Biome**: 2-space indent, single quotes, organize imports on assist (`biome.json`)
- **CSS**: Tailwind CSS v4 with `@import "tailwindcss"` (no tailwind.config.js), FlyonUI plugin with light/dark themes
- **Svelte 5**: Uses `$state()`, `$derived()`, `$props()`, `{#snippet}` / `{@render children?.()}`, `onclick` (no `on:click`). No `$effect` usage found.
- **No tests** — no test framework configured
- **Secrets required** (wrangler.jsonc): `FRONTEND_URL`, `AUTH_GOOGLE_ID`, `AUTH_GOOGLE_SECRET`
- **Auth flow**: Google OAuth → callback sets `auth_session` HttpOnly cookie in KV → worker validates on `/private/*` and `/auth/*` paths
- **Assets**: Static assets in `public/` (favicon, PWA icons, logo). Worker is also the asset server with SPA fallback.
- **PWA**: Service worker via `vite-plugin-pwa` with workbox, runtime cache for images
